mUzima

On successfully logging into the App, you'll see the mUzima dashboard. The dashboard consist of :

  1. Cohorts
  2. Clients
  3. Forms
  4. Notifications

Cohorts:
A cohort is a list of patients. This can be a static list of patient that never change (like a cohort of patients in a study) or a dynamic list of patient that is regenerated every time (like a cohort of 'all male patients over 65 years in age').

Clients:
Is a list of all patients that make up the different cohorts downloaded. A new patient can also be registered by clicking the add patient button.

Forms:
Lists all forms that have been downloaded, offers the option to download other forms as user may require. Shows completed and incompleted forms to allow the user decide appropriately. Once the user clicks on the complete form data menu, an upload option shows up and complete forms can be uploaded.

Notifications:
Lists all message communications between the device user and the server. The messages are grouped as:

  • Client messages - specific to a client
  • General messages -which target the user.
